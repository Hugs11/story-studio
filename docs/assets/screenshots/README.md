# Screenshot Guidelines

This folder is reserved for the small curated screenshot set used by the GitHub README.

## Selection Rules

- Keep only screenshots that help a new visitor understand the product quickly.
- Prefer stable views: editor, pack structure, options, emulator, and generation queues.
- Avoid personal paths, private project names, local service URLs, or generated media that cannot be redistributed.
- Use descriptive kebab-case names such as `editor-dark.png` or `options-light.png`.
- Keep files reasonably small. Prefer a few high-signal images over a large gallery.

Screenshots used only during UI review or local QA should stay outside the public repository.

## Recommended README Set

When fresh screenshots are ready, prefer this minimal set:

- `editor-dark.png` — main editing experience
- `diagram-simulator-dark.png` — full diagram with floating simulator
- `media-explorer-dark.png` — media library and asset usage
- `audio-editor-dark.png` — waveform editing workflow
- `settings-light.png` — settings and local integrations
- `home-dark.png` — optional home screen / recent projects
